# 판타지 슈팅 게임 프로젝트 — 전체 구성 지시서

> **§0. 게임 메타 정보** (CodeChecker 의사결정 힌트 — 첫 줄에 짧게)
>
> - **장르**: 판타지 종스크롤 슈팅 (single-player, 키보드 조작)
> - **레퍼런스**: 마성전설(Knightmare) 오마주. Demo는 React Canvas로 구현.
> - **best score 의미**: **Yes** — `/api/v1/game/best` endpoint 사용.
> - **장기 상태 변수**: HP, MP, 현재 stage, score, currentWeapon
> - **플레이 시간**: 약 3~10분 (HP 0 도달 시 종료)
> - **저장 단위**: 한 판 끝나면 결과 1행 INSERT

> ⚠️ **이 명세서의 가장 중요한 룰**: 모든 *수치·enum·색·매핑*은 `FE/src/constants/game.js` placeholder에 *이미 정의됨*. `GamePage.jsx`는 **반드시 import해서 사용**. 인라인 hex / 인라인 enum / 단일 숫자 fallback (`|| 10` 등) **금지**. 표 형식은 *사람이 읽기 위함*이지 LLM이 다시 변환하라는 게 아님 — placeholder가 ground truth.

## 1. 프로젝트 개요

- 마성전설(Knightmare) 오마주 판타지 종스크롤 슈팅 게임.
- 주인공이 SelectPage에서 무기 1개를 선택 → GamePage에서 *그 무기로만* 진행 (게임 중 무기 변경 불가).
- HP/MP 자원 관리 + 적 무기별 패턴 + 스테이지별 시각적 난이도 곡선이 핵심.
- Demo는 React Canvas로 1000×750 화면에서 진행.

## 4. MySQL 스키마 (테이블 2개 + 시드)

`BE/db/migrations/<UTC ts>_<name>.sql` emit. orchestrator Phase 2.5가 자동 적용. **charset/collation은 `utf8mb4` / `utf8mb4_unicode_ci`** (한글 player_name). `CREATE INDEX IF NOT EXISTS`는 MySQL 미지원 — INDEX는 `CREATE TABLE` 안에 선언.

### 4-1. `player_users` (모든 게임 완전 고정 — 수정 금지)

| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | |
| username | VARCHAR(32) UNIQUE NOT NULL | 영문/숫자/언더스코어, 4~16자 (rules/domain.md §2) |
| password_hash | CHAR(60) NOT NULL | bcrypt rounds=10 (60자) |
| player_name | VARCHAR(60) NOT NULL | 한글 허용, 2~12자 |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | |

### 4-2. `game_results` (고정 컬럼 + 게임별 컬럼 분리)

**모든 게임 고정** (game/result + game/best 표준 동작 보장):

| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | |
| player_id | BIGINT NOT NULL | FK → player_users(id) |
| score | INT NOT NULL | |
| play_time | DATETIME NOT NULL | MySQL `'YYYY-MM-DD HH:MM:SS'`. FE가 ISO 8601 보내면 BE에서 `new Date(s).toISOString().slice(0,19).replace('T',' ')`로 변환 |
| is_best | TINYINT(1) DEFAULT 0 | 개인 최고 (game/best 사용 안 해도 컬럼은 유지) |
| INDEX | (player_id, score DESC) | |

**이 게임 추가 컬럼**:

| 컬럼 | 타입 | 비고 |
|---|---|---|
| stage | TINYINT NOT NULL | 1~5 |
| weapon_used | VARCHAR(32) NOT NULL | enum: §7-10 `WEAPONS[*].id` |
| play_duration | INT NOT NULL | 초 |
| pattern_seed | VARCHAR(64) | nullable |

### 4-3. 시드 (INSERT IGNORE 1건씩 — D69 시드 정합성 필수)

- `player_users`: `(username='demo_user', password_hash='$2b$10$fmKqy9chmRPcdIq6nEbRoeP7bNX7CPQmZ7felu0VOleDBqeUO96Aq', player_name='데모용사')`
  - ⚠️ password_hash는 위 60자 *그대로* (`bcrypt.compareSync('Pass1234', hash) === true` 검증된 실제 hash).
- `game_results`: `(player_id=1, stage=1, score=500, weapon_used='holy_sword', play_duration=120, pattern_seed='seed_001', is_best=1, play_time=NOW())`

> ⚠️ **시드 정합성**: 시드 row가 BE 핸들러 validation을 *전부* 통과해야 함. signup의 username regex가 `demo_user` 통과 (underscore 허용 필수). 게임별 enum/range validator도 시드 row를 통과시킬 것.

## 5. 회원가입 / 로그인 규칙 (모든 게임 동일 규칙 — rules/domain.md §2 참조)

> 이 섹션은 게임 무관 표준. 게임별로 *내용 수정 금지*.

### 5-1. 입력 필드 (`MemberShipPage.jsx`)

| 필드 | FE 검증 (실시간) | BE 재검증 |
|---|---|---|
| ID (`username`) | `^[a-zA-Z0-9_]{4,16}$`, **/auth/check로 중복 확인 필수** | 정규식 + UNIQUE |
| Password | 8자 이상, 영문+숫자 혼합 (`^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}$`) | 동일 정규식 |
| Confirm Password | Password와 완전 일치해야 가입 버튼 활성 | (FE only) |
| Player Name | 한글/영문/숫자, 2~12자 | 길이 검증 |

### 5-2. 가입 버튼 활성 조건 (FE, AND)

1. 모든 필드 클라이언트 검증 PASS
2. Confirm Password = Password
3. **`/auth/check` 응답 `exists:false`로 중복확인 통과**

ID 입력이 바뀌면 *중복확인 상태 reset* — 다시 누르기 전까진 가입 비활성.

### 5-3. 보안

- 평문 저장 금지. **`bcrypt`** (`bcryptjs` 아님) `bcrypt.hash(password, 10)`.
- 로그인 응답에 토큰 X. `player_id` / `username` / `player_name`만.

### 5-4. BE Endpoint 4종 (+ game/best 옵션)

| # | Endpoint | Method | 핵심 | 옵션 |
|---|---|---|---|---|
| 1 | `/api/v1/auth/signup` | POST | 회원가입 | 모든 게임 필수 |
| 2 | `/api/v1/auth/check` | GET | username 중복 확인 | 모든 게임 필수 |
| 3 | `/api/v1/auth/login` | POST | bcrypt.compare + 세션 정보 | 모든 게임 필수 |
| 4 | `/api/v1/game/result` | POST | 결과 저장 + is_best 계산 | 모든 게임 필수 |
| 5 | `/api/v1/game/best` | GET | 개인 최고 + 이력 없으면 default | **이 게임 사용** (best score 의미 있음) |

`base_url` 반드시 `/api/v1` 고정 (Nginx reverse proxy 정책).

#### 5-5-1. `POST /api/v1/auth/signup` (반드시 이 순서)

1. body `{username, password, player_name}` 누락 → `400 {success:false, error:"Missing required fields"}`
2. **username**: `validateUsername(username)` (BE/src/validators.js) → 실패 `400 {error:"Invalid username format"}`
3. **password**: `validatePassword(password)` → 실패 `400 {error:"Password too short or invalid"}`
4. **player_name**: `validatePlayerName(player_name)` → 실패 `400 {error:"Invalid player name"}`
5. `SELECT id FROM player_users WHERE username=?` → 있으면 `409 {error:"Username already exists"}`
6. `bcrypt.hash(password, 10)`
7. `INSERT INTO player_users (username, password_hash, player_name) VALUES (?, ?, ?)`
8. `201 {success:true, data:{id:insertId, username, player_name}}`

#### 5-5-2. `GET /api/v1/auth/check?username=<string>`

1. query.username 누락/형식 위반 → `400 {error:"Invalid username"}`
2. `SELECT id FROM player_users WHERE username=?`
3. `200 {success:true, data:{exists:boolean}}`

#### 5-5-3. `POST /api/v1/auth/login`

1. body `{username, password}` 누락 → `400`
2. `SELECT id, username, password_hash, player_name FROM player_users WHERE username=?`
3. row 없으면 → `401 {error:"Invalid username or password"}` (계정 존재 누설 방지)
4. `bcrypt.compare(password, row.password_hash)` false → 같은 `401`
5. `200 {success:true, data:{player_id, username, player_name}}`

#### 5-5-4. `POST /api/v1/game/result` (⚠️ datetime 변환 필수)

1. body `{player_id, stage, score, weapon_used, play_duration, pattern_seed, play_time}` 누락 → `400`
2. `validateStage(stage)` (1~5) / `validateNonNegativeInt(score)` / `validateNonNegativeInt(play_duration)` / weapon_used ∈ §7-10 `WEAPONS[*].id` enum 검증 → 위반 `400`
3. **`play_time` 변환**: `new Date(play_time).toISOString().slice(0,19).replace('T',' ')` 안 하면 `ER_TRUNCATED_WRONG_VALUE` 500
4. `SELECT MAX(score) FROM game_results WHERE player_id=?` (없으면 0)
5. `score > currentBest`면 `is_best=1` + 기존 is_best=1 row를 `UPDATE ... SET is_best=0`
6. `INSERT INTO game_results (...) VALUES (...)`
7. `201 {success:true, data:{result_id:insertId, is_best}}`

#### 5-5-5. `GET /api/v1/game/best?player_id=<integer>` (옵션 — game/best 사용 시)

1. query.player_id 누락/숫자 아님 → `400`
2. `SELECT score, stage, weapon_used FROM game_results WHERE player_id=? AND is_best=1 ORDER BY score DESC LIMIT 1`
3. row 없으면 → `200 {success:true, data:{best_score:0, stage_reached:0, weapon_used:'none'}}` (null 금지 — contract type 유지)
4. row 있으면 → `200 {success:true, data:{best_score:row.score, stage_reached:row.stage, weapon_used:row.weapon_used}}`

⚠️ **시나리오 안정성** — `game/result` 시나리오 score는 *시드 best 미만* (예: 100, 200). `game/best`는 시드 그대로 검증.

### 5-6. 중복확인 FE 흐름

1. ID 입력 → 클라이언트 검증 PASS → "중복확인" 버튼 enabled
2. 클릭 → `GET /api/v1/auth/check?username=<encoded>`
3. `exists:false` → 사용 가능 + `usernameAvailable=true`
4. `exists:true` → 이미 사용 중 + `usernameAvailable=false`
5. ID 변경 시 `usernameChecked=false` reset

## 7. 게임 규칙 (가변 — 게임별 모두 다름)

### 7-1. 장르·분류

판타지 종스크롤 슈팅. SelectPage에서 선택한 *단일 무기*로 5스테이지 진행. 한 발 즉사 적. 시각적 난이도 곡선.

### 7-2. 시각 형태 — 적 모양·색 1:1 매핑

캔버스 1000×750. 주인공 흰색 원, 적·장애물·아이템·UI는 다음 매핑 표 그대로:

| 적이 든 무기 | 모양 | 색 | placeholder |
|---|---|---|---|
| `holy_sword` | 세모 (정삼각형 ▲) | `#FFD700` | `ENEMY_SHAPE_BY_WEAPON.holy_sword === 'triangle'` |
| `thunder_lance` | 별 (5각 ★) | `#A347FF` | `ENEMY_SHAPE_BY_WEAPON.thunder_lance === 'star'` |
| `fire_magic` | 네모 (정사각형 ■) | `#FF3B30` | `ENEMY_SHAPE_BY_WEAPON.fire_magic === 'square'` |
| `holy_light` | 역삼각형 (▽) | `#33E0E0` | `ENEMY_SHAPE_BY_WEAPON.holy_light === 'inverted_triangle'` |

**구현 패턴 (반드시 이 패턴 그대로)**:

```js
import { WEAPONS, ENEMY_SHAPE_BY_WEAPON } from '../constants/game';

function drawEnemy(ctx, enemy) {
  const weaponDef = WEAPONS.find(w => w.id === enemy.weapon);
  ctx.fillStyle = weaponDef.color;  // 무기 색 그대로
  const shape = ENEMY_SHAPE_BY_WEAPON[enemy.weapon];
```

### 7-6. 게임오버 조건 (frame 단위 명시)

`s.player.hp <= 0`이 되는 *그 frame*에:

1. 같은 frame 안에 `navigate('/result')` 호출. setTimeout 금지.
2. game loop 즉시 중단 (`cancelAnimationFrame` + state flag `s.gameOver=true`).
3. 진행 중인 적 spawn / 탄환 update / MP 회복 모두 중단 (gameOver flag 체크로 skip).
4. ResultPage에 마지막 state 전달 (score / stage / weapon / playDuration).

```js
import { ENEMY_POOLS, WEAPONS, ENEMY_SHAPE_BY_WEAPON, ENEMY_MOVEMENT, CANVAS } from '../constants/game';

  const weaponId = pool[Math.floor(Math.random() * pool.length)];

    y: -20,

    spawnTime: now,
    lastFireTime: now,
```

**spawn 보충 코드 (`SPAWN_CONFIG` 사용)**:

```js
import { SPAWN_CONFIG } from '../constants/game';

function updateSpawn(s, now) {
  if (s.gameOver || s.paused) return;
  const interval = SPAWN_CONFIG.intervalByStage[s.currentStage];
  const aliveCount = s.enemies.filter(e => e.alive).length;
  if (aliveCount < SPAWN_CONFIG.minOnScreen) {
    s.spawnTimer = Math.min(s.spawnTimer, now);  // 강제 spawn
  }
  if (now >= s.spawnTimer && aliveCount < SPAWN_CONFIG.maxOnScreen) {
    s.enemies.push(spawnEnemy(s, now));
    s.spawnTimer = now + interval;
  }
}
```

#### 이동·공격 패턴

| 적 | 이동 (스폰 시 3종 무작위) | 공격 주기(ms) | 조준 방식 | 탄환 속성 |
|---|---|---|---|---|
| 성검 적 (세모) | 직선 80 / 지그재그 70 / 편대 70 px/s | **1200** | 발사 시점 직선 (`single`) | 일반 |
| 번개 적 (별) | 동일 | **1500** | 발사 시점 직선 (`pierce`) | **`pierce: true` 필수** |
| 불꽃 적 (네모) | 동일 | **2000** | 발사 시점 직선 (`fan` ±20°) | 일반 |
| 성광 적 (역삼각형) | 동일 | **3000** | 발사 시점 8방향 (`omni`) | 일반 |

이동 패턴 상세 (placeholder `ENEMY_MOVEMENT`):
1. **직선 하강** (`straight`): 화면 상단에서 일정 속도로 아래로 (속도 80 px/s).
2. **지그재그 하강** (`zigzag`): 좌우 sin 곡선 (진폭 80px, 주파수 1Hz, 평균 하강속도 70 px/s).
3. **타원 편대 하강** (`formation`): 3마리 줄지어 타원 궤적 진입 후 하강 (평균 70 px/s).

**이동 코드**:

```js
import { ENEMY_MOVEMENT } from '../constants/game';
function moveEnemy(e, dt, now) {
  if (e.movement === 'straight') {
    e.y += ENEMY_MOVEMENT.straight.speed * dt;
  } else if (e.movement === 'zigzag') {
    e.y += ENEMY_MOVEMENT.zigzag.speed * dt;
    const ms = now - e.spawnTime;
    e.x += Math.sin(ms / 1000 * 2 * Math.PI * ENEMY_MOVEMENT.zigzag.frequencyHz) * ENEMY_MOVEMENT.zigzag.amplitude * dt;
  } else if (e.movement === 'formation') {
    e.y += ENEMY_MOVEMENT.formation.speed * dt;
    // 타원 궤적은 진입 1초 동안만, 이후 직선
  }
}
```

**적 발사 코드** (`ENEMY_FIRE_COOLDOWN_MS` 사용):

```js
import { ENEMY_FIRE_COOLDOWN_MS, BULLET_SPEED } from '../constants/game';

function updateEnemyFire(s, now) {
  for (const e of s.enemies) {
    if (!e.alive) continue;
    const cooldown = ENEMY_FIRE_COOLDOWN_MS[e.weapon];
    if (now - e.spawnTime < 500) continue;       // 스폰 후 500ms 유예
    if (now - e.lastFireTime < cooldown) continue;
    e.lastFireTime = now;
    // 발사 시점 dx/dy 1회만 (homing 금지)
    const dx = s.player.x - e.x;
    const dy = s.player.y - e.y;
    const len = Math.hypot(dx, dy);
    const vx = (dx/len) * BULLET_SPEED;
    const vy = (dy/len) * BULLET_SPEED;
    s.enemyBullets.push({ x: e.x, y: e.y, vx, vy, color: e.color, pierce: e.weapon === 'thunder_lance' });
  }
}
```

**필수 룰**:
1. 발사 주기는 표 ms 값 *그대로*. *적 발사 주기(1200~3000ms)*와 *주인공 발사 딜레이(100~400ms, §7-4)*는 서로 다른 개념.
2. 번개 적탄 `pierce:true` — 플레이어 충돌 시 HP만 깎고 탄환 *유지*. 일반 탄환은 소멸.
3. 탄환 색 = §7-4 무기 색 HEX 그대로.
4. 스폰 후 **0.5초 유예** 뒤 첫 발사.
5. 적 발사 분기 라벨은 주인공 라벨과 동일 (`single/fan/pierce/omni`) — *플레이어/적 공통 helper* 권장.

#### 피격·사망

적은 **플레이어 탄환에 한 발이라도 맞으면 즉시 사망**. 적은 HP 개념 *없음*. `bullet ↔ enemy` 충돌 감지 시:

1. 해당 적을 즉시 제거 (`enemy.alive = false`)
2. 탄환도 제거 (단 *주인공* `pierce:true` 탄환은 예외 — §7-4. 적의 `pierce`는 *주인공* 피격에만 적용, 적 자신은 *항상 즉사*)
3. score 가산 (§7-8 stage별)
4. `SPAWN_CONFIG` 보충 룰에 따라 *다음 spawn tick*에서 자동 충원

**코드**:

```js
import { SCORE_PER_STAGE, STAGES } from '../constants/game';

function handleBulletEnemyCollision(s, bullet, enemy) {
  enemy.alive = false;
  if (!bullet.pierce) bullet.dead = true;
  s.score += SCORE_PER_STAGE[s.currentStage];
  // §7-5 stage 전환 검사
  const cur = STAGES[s.currentStage - 1];
  if (cur.scoreThreshold !== null && s.score >= cur.scoreThreshold && s.currentStage < 5) {
    s.currentStage += 1;
  }
}
```

- 🚫 **금지**: HP/durability/무적시간 (한 발 즉사 룰 위반). `enemy.hp`, `enemy.durability`, `enemy.invuln*` 필드 0개.

#### 🚫 적 행동 금지 패턴 (필수 — LLM이 자주 어김)

- **유도탄 (homing)**: 매 frame `dx = player.x - bullet.x` 재계산. 절대 금지.
- **추적 (tracking)**: setInterval로 주기적 vx/vy 갱신. 금지.
- **곡선 궤적**: sin/cos 적용한 휘어진 경로. 금지 (단 `holy_light` 8방향 단발은 *각도 8개 고정* 직선 OK).
- **속도 임의 변형**: 표 외 속도 (예: 400 px/s 같은 임의 값) 금지.
- 모든 적을 동일 모양(예: 모두 세모)으로 그리는 구현 — §7-2 4종 모양 모두 등장 필수.

### 7-12. 표 → 코드 매핑 강제 (핵심 룰)

본 명세서의 *모든 표*는 §7-10 placeholder에 이미 정의됨. GamePage.jsx는 **표 내용을 다시 적지 말고 placeholder import만**.

- 표는 *사람이 읽기 위함*.
- 코드는 *placeholder 그대로 사용*.
- LLM이 표를 다시 JS 객체로 변환하지 말 것. 이미 §7-10에 정의돼 있음.
- *인라인 변환*하면 SpecSync는 통과해도 시각적 회귀.

### 9-2. `report.html` 다운로드 (Blob + download attribute)

```jsx
const html = `<!DOCTYPE html><html><body><h1>Result</h1>...</body></html>`;

const url = URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url; a.download = 'report.html';
a.click();
URL.revokeObjectURL(url);
```
