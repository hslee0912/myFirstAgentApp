# 판타지 슈팅 게임 프로젝트 — 전체 구성 지시서

> **§0. 게임 메타 정보**
>
> - **장르**: 판타지 종스크롤 슈팅 (single-player, 키보드 조작)
> - **레퍼런스**: 마성전설(Knightmare) 오마주. Demo는 React Canvas로 구현.
> - **best score 의미**: **Yes** — `/api/v1/game/best` endpoint 사용.
> - **장기 상태 변수**: HP, MP, 현재 stage, score, currentWeapon
> - **플레이 시간**: 약 3~10분 (HP 0 도달 시 종료)
> - **저장 단위**: 한 판 끝나면 결과 1행 INSERT

> ⚠️ **이 명세서의 가장 중요한 룰**: 모든 *수치·enum·색·매핑*은 `FE/src/constants/game.js` placeholder에 *이미 정의됨*. `GamePage.jsx`는 **반드시 import해서 사용**. 인라인 hex / 인라인 enum / 단일 숫자 fallback (`|| 10` 등) **금지**. 표 형식은 *사람이 읽기 위함*이지 LLM이 다시 변환하라는 게 아님 — placeholder가 ground truth.

## 1. 프로젝트 개요

- 마성전설(Knightmare) 오마주 판타지 종스크롤 슈팅 게임.
- 주인공이 SelectPage에서 무기 1개를 선택 → GamePage에서 *그 무기로만* 진행 (게임 중 무기 변경 불가).
- HP/MP 자원 관리 + 적 무기별 패턴 + 스테이지별 시각적 난이도 곡선이 핵심.
- Demo는 React Canvas로 1000×750 화면에서 진행.

## 4. MySQL 스키마 (테이블 2개 + 시드)

`BE/db/migrations/<UTC ts>_<name>.sql` emit. orchestrator Phase 2.5가 자동 적용. **charset/collation은 `utf8mb4` / `utf8mb4_unicode_ci`** (한글 player_name). `CREATE INDEX IF NOT EXISTS`는 MySQL 미지원 — INDEX는 `CREATE TABLE` 안에 선언.

### 4-1. `player_users` (완전 고정)

| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | |
| username | VARCHAR(32) UNIQUE NOT NULL | 영문/숫자/언더스코어, 4~16자 |
| password_hash | CHAR(60) NOT NULL | bcrypt rounds=10 |
| player_name | VARCHAR(60) NOT NULL | 한글 허용, 2~12자 |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | |

### 4-2. `game_results`

| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | |
| player_id | BIGINT NOT NULL | FK → player_users(id) |
| score | INT NOT NULL | |
| stage | TINYINT NOT NULL | 1~5 |
| weapon_used | VARCHAR(32) NOT NULL | enum: §7-10 `WEAPONS[*].id` |
| play_duration | INT NOT NULL | 초 |
| pattern_seed | VARCHAR(64) | nullable |
| is_best | TINYINT(1) DEFAULT 0 | |
| play_time | DATETIME NOT NULL | MySQL `'YYYY-MM-DD HH:MM:SS'`. FE가 ISO 8601 보내면 BE에서 `new Date(s).toISOString().slice(0,19).replace('T',' ')` |
| INDEX | (player_id, score DESC) | |

### 4-3. 시드

- `player_users`: `(username='demo_user', password_hash='$2b$10$fmKqy9chmRPcdIq6nEbRoeP7bNX7CPQmZ7felu0VOleDBqeUO96Aq', player_name='데모용사')`

## 5. 회원가입 / 로그인 규칙 (모든 게임 동일 — rules/domain.md §2 참조)

### 5-1. 입력 필드 (`MemberShipPage.jsx`)

| 필드 | FE 검증 | BE 재검증 |
|---|---|---|
| ID (`username`) | `^[a-zA-Z0-9_]{4,16}$`, **/auth/check 중복 확인 필수** | 정규식 + UNIQUE |
| Password | 8자 이상, 영문+숫자 혼합 (`^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}$`) | 동일 |
| Confirm Password | Password와 완전 일치 | (FE only) |
| Player Name | 한글/영문/숫자, 2~12자 | 길이 검증 |

### 5-2. 가입 버튼 활성 조건 (FE, AND)
1. 모든 필드 클라이언트 검증 PASS
2. Confirm Password = Password
3. `/auth/check` 응답 `exists:false`로 중복확인 통과

ID 입력 변경 시 중복확인 상태 reset.

### 5-3. 보안
- 평문 저장 금지. `bcrypt.hash(password, 10)` (`bcryptjs` 아님).
- 로그인 응답에 토큰 X. `player_id` / `username` / `player_name`만.

### 5-4. BE Endpoint 5종

| # | Endpoint | Method |
|---|---|---|
| 1 | `/api/v1/auth/signup` | POST |
| 2 | `/api/v1/auth/check` | GET |
| 3 | `/api/v1/auth/login` | POST |
| 4 | `/api/v1/game/result` | POST |
| 5 | `/api/v1/game/best` | GET |

#### 5-5-1. `POST /api/v1/auth/signup`
1. body `{username, password, player_name}` 누락 → `400 {error:"Missing required fields"}`
2. `validateUsername(username)` 실패 → `400 {error:"Invalid username format"}`
3. `validatePassword(password)` 실패 → `400 {error:"Password too short or invalid"}`
4. `validatePlayerName(player_name)` 실패 → `400 {error:"Invalid player name"}`
5. `SELECT id FROM player_users WHERE username=?` → 있으면 `409 {error:"Username already exists"}`
6. `bcrypt.hash(password, 10)` → INSERT → `201 {success:true, data:{id, username, player_name}}`

#### 5-5-2. `GET /api/v1/auth/check?username=<string>`
1. 누락/형식 위반 → `400 {error:"Invalid username"}`
2. SELECT → `200 {data:{exists:boolean}}`

#### 5-5-3. `POST /api/v1/auth/login`
1. 누락 → `400`
2. row 없거나 bcrypt.compare false → `401 {error:"Invalid username or password"}`
3. PASS → `200 {data:{player_id, username, player_name}}`

#### 5-5-4. `POST /api/v1/game/result` (datetime 변환 필수)
1. body 누락 → `400`
2. `validateStage` (1~5) / `validateNonNegativeInt(score/play_duration)` / weapon_used ∈ §7-10 `WEAPONS[*].id` enum → 위반 `400`
3. `play_time` 변환: `new Date(play_time).toISOString().slice(0,19).replace('T',' ')`
4. `SELECT MAX(score) FROM game_results WHERE player_id=?` (없으면 0)
5. score > currentBest면 is_best=1 + 기존 is_best=1 row UPDATE 0
6. INSERT → `201 {data:{result_id, is_best}}`

#### 5-5-5. `GET /api/v1/game/best?player_id=<int>`
1. 누락/숫자 아님 → `400`
2. row 없으면 → `200 {data:{best_score:0, stage_reached:0, weapon_used:'none'}}` (null 금지)
3. row 있으면 → `200 {data:{best_score, stage_reached, weapon_used}}`

> PostTest 안정성: game/result 시나리오 score 모두 시드 500 미만. game/best는 시드 검증.

### 5-6. 중복확인 FE 흐름
1. ID 입력 + 클라이언트 검증 PASS → "중복확인" 버튼 enabled
2. 클릭 → GET /auth/check → exists:false → `usernameAvailable=true`
3. ID 변경 시 `usernameChecked=false` reset

## 7. 게임 규칙

### 7-1. 장르
판타지 종스크롤 슈팅. SelectPage에서 선택한 *단일 무기*로 5스테이지 진행. 한 발 즉사 적.

### 7-2. 적 모양·색 1:1 매핑

| 적이 든 무기 | 모양 | 색 | placeholder |
|---|---|---|---|
| `holy_sword` | 세모 (정삼각형 ▲) | `#FFD700` | `ENEMY_SHAPE_BY_WEAPON.holy_sword === 'triangle'` |
| `thunder_lance` | 별 (5각 ★) | `#A347FF` | `ENEMY_SHAPE_BY_WEAPON.thunder_lance === 'star'` |
| `fire_magic` | 네모 (정사각형 ■) | `#FF3B30` | `ENEMY_SHAPE_BY_WEAPON.fire_magic === 'square'` |
| `holy_light` | 역삼각형 (▽) | `#33E0E0` | `ENEMY_SHAPE_BY_WEAPON.holy_light === 'inverted_triangle'` |

**구현 패턴 (반드시 이 패턴 그대로)**:

```js
import { WEAPONS, ENEMY_SHAPE_BY_WEAPON } from '../constants/game';

function drawEnemy(ctx, enemy) {
  const weaponDef = WEAPONS.find(w => w.id === enemy.weapon);
  ctx.fillStyle = weaponDef.color;  // 무기 색 그대로
  const shape = ENEMY_SHAPE_BY_WEAPON[enemy.weapon];
  if (shape === 'triangle')          { /* moveTo+lineTo 정삼각형 */ }
  else if (shape === 'star')         { /* 10점 별 */ }
  else if (shape === 'square')       { ctx.fillRect(enemy.x-12, enemy.y-12, 24, 24); }
  else if (shape === 'inverted_triangle') { /* 역삼각형 */ }
}
```

🚫 **금지**: `enemy.color = '#ff4466'` 같은 인라인 hex. 모든 적 색은 *반드시* `WEAPONS.find(...).color`.

### 7-4. 자원·점수·발사 룰 — *코드 패턴 반드시 적용*

#### MP 회복 (필수 — *placeholder 속성명 그대로*)

```js
import { MP_REGEN, HERO_INITIAL } from '../constants/game';

// game loop 매 frame 안 (gameOver / paused 아닐 때만):
if (!s.gameOver && !s.paused) {
  const now = Date.now();
  const sinceLastAttack = now - s.lastAttackTime;
  const sinceLastRegen  = now - s.lastMpRegen;
  // 공격 후 cooldownAfterFireMs 동안은 회복 X, 그 외엔 intervalMs마다 amount씩
  if (sinceLastAttack >= MP_REGEN.cooldownAfterFireMs && sinceLastRegen >= MP_REGEN.intervalMs) {
    s.player.mp = Math.min(HERO_INITIAL.mp, s.player.mp + MP_REGEN.amount);
    s.lastMpRegen = now;
  }
}
```

⚠️ **placeholder 속성명 정확**: `MP_REGEN.cooldownAfterFireMs` / `MP_REGEN.intervalMs` / `MP_REGEN.amount`. *다른 이름으로 변형 금지* (`attackCooldownMs` 같은 변형 발견 시 placeholder import 누락 또는 잘못된 이름 사용 — 명세 위반).

#### HP 감소 (피탄·충돌)

```js
import { DAMAGE } from '../constants/game';
// 적 탄환 피탄 시
s.player.hp = Math.max(0, s.player.hp - DAMAGE.bulletHit);
// 적 본체 충돌 시
s.player.hp = Math.max(0, s.player.hp - DAMAGE.enemyCollide);
```

#### 발사 (Space 키 입력)

```js
import { WEAPONS } from '../constants/game';

function tryFire(s, now) {
  const wId = s.player.weapon;
  const wDef = WEAPONS.find(w => w.id === wId);
  if (s.player.mp < wDef.cost) return;  // MP 부족
  if (now - (s.lastFireTime[wId] || 0) < wDef.fireDelayMs) return;  // cooldown
  s.player.mp = Math.max(0, s.player.mp - wDef.cost);
  s.lastFireTime[wId] = now;
  s.lastAttackTime = now;  // MP 회복 cooldown trigger
  spawnPlayerBullet(s, wDef);  // wDef.pattern: single/fan/pierce/omni 분기
}
```

#### 주인공 무기 4종 (placeholder `WEAPONS` 그대로)

| 무기 (한글) | id | cost | 패턴 | 색 | 발사 딜레이 |
|---|---|---|---|---|---|
| 성검 | `holy_sword` | 0 | single (직선 단발) | `#FFD700` | 100ms |
| 번개 창 | `thunder_lance` | 3 | pierce (관통 고속) | `#A347FF` | 300ms |
| 불꽃 마법 | `fire_magic` | 5 | fan (부채꼴 3발 ±20°) | `#FF3B30` | 200ms |

#### 7-9-1. spawn 시 무기 결정 (필수 — ENEMY_POOLS 사용 강제)

```js
import { ENEMY_POOLS, WEAPONS, ENEMY_SHAPE_BY_WEAPON, ENEMY_MOVEMENT } from '../constants/game';

function spawnEnemy(s, now) {
  const pool = ENEMY_POOLS[s.currentStage];  // ← stage별 pool 사용
  const weaponId = pool[Math.floor(Math.random() * pool.length)];
  const weaponDef = WEAPONS.find(w => w.id === weaponId);
  const movements = ['straight', 'zigzag', 'formation'];
  const movement = movements[Math.floor(Math.random() * movements.length)];
  return {
    x: Math.random() * (CANVAS.width - 40) + 20,
    y: -20,
    weapon: weaponId,
    color: weaponDef.color,
    shape: ENEMY_SHAPE_BY_WEAPON[weaponId],
    movement,
    spawnTime: now,
    lastFireTime: now,
    alive: true,
  };
}
```

🚫 **금지 (이번 cycle 실패 원인)**:
- `enemy.weapon = 'holy_sword'` 같이 하나로 고정 → 모든 적이 디폴트 무기
- 랜덤 hex 색 사용 → 적 색 ≠ 무기 색
- ENEMY_POOLS 안 쓰기 → stage별 적 다양성 X

#### 7-9-2. spawn 보충

```js
import { SPAWN_CONFIG } from '../constants/game';

function updateSpawn(s, now) {
  if (s.gameOver || s.paused) return;
  const interval = SPAWN_CONFIG.intervalByStage[s.currentStage];
  const aliveCount = s.enemies.filter(e => e.alive).length;
  // 화면 적이 minOnScreen 미만이면 즉시 spawn 강제
  if (aliveCount < SPAWN_CONFIG.minOnScreen) {
    s.spawnTimer = Math.min(s.spawnTimer, now);
  }
  if (now >= s.spawnTimer && aliveCount < SPAWN_CONFIG.maxOnScreen) {
    s.enemies.push(spawnEnemy(s, now));
    s.spawnTimer = now + interval;
  }
}
```

#### 7-9-3. 이동 패턴 (placeholder `ENEMY_MOVEMENT` 사용)

```js
function moveEnemy(e, dt, now) {
  if (e.movement === 'straight') {
    e.y += ENEMY_MOVEMENT.straight.speed * dt;
  } else if (e.movement === 'zigzag') {
    e.y += ENEMY_MOVEMENT.zigzag.speed * dt;
    const ms = now - e.spawnTime;
    e.x += Math.sin(ms / 1000 * 2 * Math.PI * ENEMY_MOVEMENT.zigzag.frequencyHz) * ENEMY_MOVEMENT.zigzag.amplitude * dt;
  } else if (e.movement === 'formation') {
    e.y += ENEMY_MOVEMENT.formation.speed * dt;
    // 타원 궤적은 진입 1초 동안만, 이후 직선
  }
}
```

#### 7-9-4. 적 발사 (placeholder `ENEMY_FIRE_COOLDOWN_MS` 사용)

```js
import { ENEMY_FIRE_COOLDOWN_MS, BULLET_SPEED } from '../constants/game';

function updateEnemyFire(s, now) {
  for (const e of s.enemies) {
    if (!e.alive) continue;
    const cooldown = ENEMY_FIRE_COOLDOWN_MS[e.weapon];
    // 첫 발사는 스폰 후 500ms 유예
    if (now - e.spawnTime < 500) continue;
    if (now - e.lastFireTime < cooldown) continue;
    e.lastFireTime = now;
    // 발사 시점 dx/dy 1회만 (homing 금지)
    const dx = s.player.x - e.x;
    const dy = s.player.y - e.y;
    const len = Math.hypot(dx, dy);
    const vx = (dx/len) * BULLET_SPEED;
    const vy = (dy/len) * BULLET_SPEED;
    s.enemyBullets.push({ x: e.x, y: e.y, vx, vy, color: e.color, pierce: e.weapon === 'thunder_lance' });
  }
}
```

🚫 **금지**:
- 매 frame `vx = (player.x - bullet.x) / len * speed` 재계산 (homing)
- setInterval로 vx/vy 갱신 (tracking)
- 표 외 속도 (예: 400 px/s) 임의 사용
